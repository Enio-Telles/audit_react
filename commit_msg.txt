⚡ Cache existing directories to avoid redundant disk I/O

💡 **What:** The `_resolver` function inside `gerar_tabela_itens_caracteristicas` checks if `arq_dir` and `pasta_cnpj` directories exist before searching for files inside them. This check (`diretorio.exists()`) was originally inside the `_resolver` function, which is called multiple times. Now, the valid directories are cached before the `_resolver` definition.
🎯 **Why:** The previous approach performed a disk operation (`diretorio.exists()`) every time `_resolver` was called. Since `_resolver` is used to find multiple parquet files (`reg_0000`, `NFe`, `NFCe`, `c170_simplificada`, `c170`, `bloco_h`), checking if the directories exist outside the loop saves multiple unnecessary I/O checks.
📊 **Measured Improvement:** We didn't measure the exact performance improvement in this case because the `tests` directory is empty and there's no benchmark suite available. However, since the `_resolver` function is called 6 times, we skip 6 unnecessary disk queries (stat system calls) for each CNPJ processed, leading to a small but cumulative performance gain, especially on slower disk drives or network-attached storage.
